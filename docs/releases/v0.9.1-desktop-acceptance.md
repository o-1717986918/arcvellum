# ArcVellum v0.9.1 Desktop Acceptance

这份清单覆盖无法由源码检查代替的 Windows 桌面验收。

## 构建前

1. 在正常 Windows 桌面环境运行 `packaging/build_desktop.ps1`。不要单独运行 `tauri build`；正式脚本会重建、记录并验证冻结 sidecar。
2. 确保构建环境具有项目声明的 Python、Node、Rust 依赖和现有 `TAURI_SIGNING_PRIVATE_KEY`。不要新建或替换更新签名密钥。
3. 确认 `build/sidecar-provenance.json` 已生成，且 `npm run desktop:verify-sidecar` 返回成功。

## 空间场域

在至少一部含多个章节、每章多个场景、人物关系、分支和审查证据的作品上验证：

1. 在全书、章节、场景视图切换。全书应显示章节主脉络；章节应显示场景承接曲线；场景应显示人物、分支、审查、Canon、承诺和任务的局部曲线。
2. 平移、缩放、聚焦节点和打开多个详情窗。曲线应平滑跟随节点，不应变成直角折线、随机换边或遮盖节点标签。
3. 切换主题、动效、纵深与渲染质量。主脉络、局部曲线、节点图标、窗口材质和背景应保持层级与对比度。
4. 开启减少动效和 WebGL 回退。所有正式项目操作仍应可达，且关系信息可通过无障碍列表阅读。
5. 为相邻场景标注连续时间、长跳时和闪回时间。长跳时应增加主脉络留白；闪回仍应位于后续阅读位置，不得使章节曲线回绕或反向。

## 安装与更新

1. 验证冷启动不闪出终端窗口，并能进入加载与正式项目页面。
2. 对 NSIS 执行新安装、覆盖升级、卸载与重启测试。
3. 通过本地 health endpoint 确认运行中的 sidecar 报告 `0.9.1`，再开始项目操作。
4. 验证更新清单、目录选择、模型连接、正文阅读、交付下载和诊断导出在安装后均可用。

## 桌面与冻结产物

正式打包完成以下步骤：

1. Vue 生产构建与单文档 Tauri 前端同步；
2. PyInstaller 6.21.0 冻结 Python 3.14 sidecar；
3. 捆绑 OpenCode 1.18.3 与许可证通知；
4. Rust release 编译；
5. Tauri NSIS x64 安装包；
6. updater 签名与 `latest.json`；
7. 当前版本发布目录清理与 SHA256 清单。

冻结的 `dist/literary-engineering-studio-sidecar.exe` 已作为独立进程启动：`/health` 返回 `ok=true`、`version=0.7.0`，打包前端首页返回 HTTP 200 并包含 ArcVellum。

打包过程还修复了两个真实发布缺口：PyInstaller 之前会先检查冻结 sidecar 是否被旧烟测进程占用；更新清单会按请求版本选择签名产物，并清理 `dist/release` 中上一版本的发布文件。

## 保留边界

- updater 使用 Tauri minisign 链，不等于 Windows Authenticode 商业代码签名；
- 本轮未覆盖安装现有用户应用，以免改变当前安装登记；冻结 sidecar 和 NSIS 产物已完成独立验证；
- 干净 Windows 10/11 的首次安装、覆盖升级、卸载、WebView2 故障与签名回滚仍是 v1.0 发布门槛；
- 50 万字用例验证阅读投影与缓存，不代替真实模型持续创作 50 万字的质量验证；
- 更长时间、多类型、无人值守的真实项目仍需继续积累创作质量和恢复证据。

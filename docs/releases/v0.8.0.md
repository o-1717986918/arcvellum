# ArcVellum v0.8.0

ArcVellum v0.8.0 将创作规则和叙事结构从后台工程能力带到普通用户可操作的产品界面。

## 新增

- Creative Quality Profile：项目级创作规则、三级强度、即时样文检查、版本摘要与场景范围例外；
- Narrative Projection v2：事件序号、投影差异和由真实状态驱动的动效事件；
- Narrative Orrery：全幅叙事星仪、章节/场景聚焦、时间轴、节点详情和无障碍列表；
- 双模式叙事星仪：工作台与全视口沉浸模式共用真实投影，边缘仪表可收起，并提供苔夜、靛紫、黑曜、书柜和现代五套完整主题；
- 沉浸工作台控制坞：自动推进、人工决策、规则与节奏、正式正文和路线健康度在全景模式中保持完整可用；原生仪器窗可多开、折叠、拖动、置顶、视口约束和复位，默认四区布局避免覆盖标题、场景选择与中央脉络；
- 五张主题专属生成式背景只承担低对比空间材质；每套主题使用主材质、互补功能色和半透明矿物层，背景不包含虚假 UI 或项目数据；
- 贯通式启动：真实 Bootstrap 阶段驱动画面，服务就绪后允许跳过视觉等待；
- 正式帮助、作品与应用详情、使用协议、隐私说明和第三方组件说明。

## 创作链路

- Compose、Generate、AgentReview、Revise、Promotion 与 Release 固化同一份创作品质档案摘要；
- 静态 Style Lint、标点检查与 AgentReview 使用同一场景范围规则；
- 修改档案只影响未来候选，旧候选必须重新生成或审查；
- 场景例外必须登记规则、范围、理由和强度，不能作为绕过工程门禁的开关。

## 可视化与性能

- 全书视图按章节聚合大型作品，1000 场景测试不会直接绘制 1000 个场景节点；
- 场景、分支、正式正文、Canon 与任务变化分别映射到可解释动画；
- 支持列表模式、键盘操作与 `prefers-reduced-motion`。
- Dashboard、Library、Reader 与 Narrative 通过项目修订指纹复用短期读模型，内容不变时 SSE 只发心跳。

## Agent Runtime

- Worker、顾问和自动审批使用角色隔离的长驻 OpenCode 服务与独立权限配置，连续任务不再重复冷启动；
- 正文、推演与审查任务会被编译成精简 Studio Worker Program，只暴露本次允许读取和写入的合同；
- 正式写回前执行确定性预检，格式或完成证据错误会在同一 Agent 会话中限次修复；
- 应用中断后仅在旧沙箱通过身份、范围与产物预检时从写回边界恢复，否则回退到新任务；
- 高频文字增量走内存事件总线，里程碑持久化；界面显示真实阶段、TTFT、修订和连接活动；
- 创作顾问支持安全 Markdown，禁用原始 HTML、远程图片和危险链接，并复用同一项目会话。
- 顾问升级为受控自然语言项目控制台，可把明确指令转成需确认的白名单动作；全自动授权仍由后端硬门禁保护。
- `/health` 不再同步等待 Runner 与内核冷探测，启动页可以立即获得真实检查进度；OpenCode 服务工作目录固定在应用数据区，避免 Windows 临时工作区被长驻进程占用。

## 升级

可从 v0.7.0 或 v0.7.1 直接覆盖安装。作品库与自定义项目目录不会被移动或删除。新建作品会自动获得默认创作品质档案；旧作品在首次读取时使用兼容默认值，保存规则后写入项目。

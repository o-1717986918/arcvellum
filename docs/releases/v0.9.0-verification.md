## Windows Desktop Bundle

Tauri compiled the release application and created the NSIS installer:

```text
ArcVellum_0.9.0_x64-setup.exe
size: 91,823,075 bytes
sha256: 3A5591E0BFB9289E3A20A92F7868959BB2549A624159B84CF1433CCA46674D16
```

The WebP-only startup and theme resources reduce the generated installer by approximately `10.4 MB` compared with the preceding package build.

The final updater-signature stage intentionally stopped because `TAURI_SIGNING_PRIVATE_KEY` is not available in the build environment. In addition, this local installer was created through a direct Tauri smoke build after the source version changed; its frozen Python sidecar predates the `0.9.0` metadata update. It confirms Tauri resource bundling only and must not be distributed.

A publishable build must run `packaging/build_desktop.ps1` on a Windows build machine that has its declared Python dependencies available. That script rebuilds the sidecar from the current source, copies it into the Tauri binary directory, creates the installer, and signs the updater with the existing project key. Do not replace the configured public key or generate a substitute key for this release.

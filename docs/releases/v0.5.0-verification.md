## 前端实机检查

使用本地 v0.5.0 服务检查：

- `390x844`：移动导航、启动场景、总控卡片和顾问入口无横向溢出。
- `1024x768`：设置、模型恢复、版本信息、正文档案和悬浮顾问可用。
- `1440x900`：主总控与 438 像素顾问面板并排，无页面横向溢出。

- 作品档案确认存在独立 `reader-scroll` 正文滚动容器。

- “关于 ArcVellum”显示版本 `0.5.0`、本地数据位置、更新通道和诊断入口。

真实顾问验证发现并修复了一个并发问题：总控 SSE 刷新的 `workflow/dashboard` 投影曾被只读完整性检查误判为作品修改。现在这些展示层投影不会进入顾问快照或项目完整性哈希；在总控持续刷新期间，顾问已成功完成 OpenCode 模型调用、自然回答和流式回传。

## 保留边界

- updater 的 minisign 签名已验证构建链路，但不是 Windows Authenticode 商业代码签名。
- 本机已有旧版本安装，为避免破坏用户当前安装登记，本轮没有运行 NSIS 覆盖安装；使用冻结 sidecar 烟测验证可执行包。干净 Windows 10/11 的首次安装、覆盖升级、卸载、WebView2 故障和签名回滚仍是 v1.0 发布门槛。
- 三章全书交付由确定性 Worker 验证调度和发布结构；真实模型以四类型高风险 Prompt 链路验证。更长时间、更多章节的无人值守真实项目仍需继续积累质量样例。

# ArcVellum v0.7.1 Verification

验证日期：2026-07-21

验证平台：Windows 11 x64

候选版本：`0.7.1`

## 根因

安装版前端从 `http://tauri.localhost` 访问随机端口上的 loopback sidecar。前端请求使用 `credentials: include`，FastAPI CORS 却没有返回 `Access-Control-Allow-Credentials`。服务实际已经启动并返回响应，但 WebView 按浏览器规则将响应拦截为 `Failed to fetch`。

开发模式通过 Vite 同源代理访问，因此没有复现。v0.7.0 的冻结产物烟测只直接请求 `/health`，也没有经过 WebView CORS。

## 修复

- 受信任 Origin 仍限定为 `http://tauri.localhost`、`https://tauri.localhost` 和 `tauri://localhost`；
- 允许这些 Origin 携带桌面会话凭据；
- 随机 Bearer 启动令牌、随机 loopback 端口和 API 鉴权保持不变；
- 前端将底层网络错误转换为中文恢复提示；
- 发布手册新增真实安装版 WebView 验证要求。

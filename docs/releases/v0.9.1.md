# ArcVellum v0.9.1

ArcVellum v0.9.1 是 v0.9 的导航与桌面发布补丁。它不改变文学工程内核、Agent Worker 或 CLI 状态机的职责边界，而是让空间场域的层级关系更可读，并阻止过期后端被打进新安装包。

## 空间叙事导航

- 全书视图继续以章节作为主脉络；
- 章节视图将场景之间的正式 `bridge` 关系绘制为高可见度、连续的平滑接续曲线；
- 场景视图将人物参与、分支、审查、Canon 写回、读者问题、承诺与当前工作流绘制为具有不同语义颜色的局部曲线；
- 画布与前景叠层共用真实投影边，不伪造关系；曲线使用稳定的三次贝塞尔控制点，因此刷新、平移与缩放不会随机折断或跳变；
- 章节、场景与关系节点均保留已有的类型图标、动画、可聚焦详情窗和无障碍列表回退。
- 星图主脉络以阅读顺序单向推进，并读取正式 `timeline_order`：故事跳时自动留出更大的空间，闪回或时间重叠只压缩间距、不让章节倒退；节奏面板可为某场设置投影专用的前置间距，不会改写正文时间。

## 桌面发布门禁

- `packaging/build_desktop.ps1` 在冻结 Python sidecar 后生成 `build/sidecar-provenance.json`；
- 打包前重新核验应用版本、当前 Python 源码哈希、冻结二进制哈希和二进制名称；
- `npm run desktop:build` 同样先执行 `npm run desktop:verify-sidecar`；缺失、过期或错配的 sidecar 会直接终止构建；
- 这避免出现“前端显示新功能、安装包仍携带旧本地服务”的静默不一致。

## 发布前验收

必须在受信任的 Windows 桌面环境执行正式打包与安装验收。详情见 [v0.9.1 desktop acceptance](v0.9.1-desktop-acceptance.md)。本仓库的类型检查、前端生产构建与 sidecar provenance 单元测试可作为源码级证据，但不替代真实 Windows 安装、启动和 WebView 视觉验收。

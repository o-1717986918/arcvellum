# ArcVellum v0.5.0

ArcVellum 0.5.0 把原先面向工程维护者的 Agent 控制台，收敛为普通创作者可以长期使用的长篇文学桌面工作台。

## 本版重点

- 全新的 Vue 3、TypeScript、Vite 客户端，以作品、正文、创作进度和交付为主要信息。
- 原生目录选择、启动状态聚合、模型后台恢复和可降级启动。
- “叙事观测台”视觉系统、动态故事脉络、冷白正文阅读面与减少动态效果支持。
- 自然流式悬浮顾问，支持长期会话摘要、固定偏好、折叠证据与安全动作建议。
- AutopilotController 提供协作、监督自动、全自动三种模式，并受任务数、时间、费用、失败和修订上限约束。
- CreativeSteward 作为独立的只读决策主体处理已授权的分支、修订、扩纲、设定和发布选择，决策身份不会冒充用户。
- 全书交付协调器汇总正式章节，过滤工作流痕迹，生成 Markdown、DOCX、检查报告和可追溯 Release Manifest。
- Prompt Asset 元数据和显式任务契约完整传入 Runner，旧任务只在兼容层明确标记后读取。
- 应用内版本、更新、数据位置和脱敏诊断中心；Tauri updater 产物启用签名验证。

## 安装与升级

下载 `ArcVellum_0.5.0_x64-setup.exe`（实际文件名以 Release 附件为准）并运行。安装包包含本地服务、文学工程内核和固定版本 OpenCode Runner，不要求预装 Python、Node.js、Rust 或独立 Agent 平台。

现有安装可以原位升级；应用 identifier 与数据目录身份保持不变。模型服务仍需要网络和用户自行选择的有效模型凭证，凭证由 OpenCode 本地认证存储管理，不写入作品或 Studio 普通配置。

## 已知边界

- updater 签名用于验证更新包来源，不等同于 Windows Authenticode 商业代码签名。
- 大规模无人值守创作仍受模型质量、上下文和费用影响；自动模式到达质量或授权边界时会暂停，而不会绕过正式门禁。
- v1.0 之前仍需继续积累多类型、长上下文和干净虚拟机的真实运行样例。

## 验证记录

本候选版本已通过 84 个 Python 测试、9 个 Vue 测试、四类型真实模型 Prompt 评测、Vue 生产构建、Rust 检查、四种视口实机检查、冻结 sidecar 启动烟测和签名更新清单验证。完整命令、指标、文件哈希与保留边界见 [v0.5.0 verification](v0.5.0-verification.md)。

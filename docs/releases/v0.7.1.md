# ArcVellum v0.7.1

ArcVellum v0.7.1 is a desktop connectivity hotfix for the Windows installer.

## Fixed

- Allow authenticated requests from the packaged Tauri WebView to the loopback application service.
- Prevent the browser from collapsing a healthy sidecar response into `Failed to fetch` because of a CORS credentials mismatch.
- Replace low-level network errors with a Chinese recovery message for ordinary users.
- Add a regression test for the packaged Tauri Origin, authorization preflight and credential response header.

## Why development mode was unaffected

Vue development uses a same-origin Vite proxy. The packaged application uses a Tauri Origin and a randomly selected loopback port, so it exercises a browser CORS path that the previous release smoke test did not cover.

## Upgrade

Install v0.7.1 over v0.7.0. Existing works under `Documents/ArcVellum/Works` and custom project locations are not moved or removed.

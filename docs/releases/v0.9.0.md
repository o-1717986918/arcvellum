# ArcVellum v0.9.0

ArcVellum v0.9.0 将叙事星仪从固定图表升级为可工作的 2.5D 叙事场域，同时不改变文学工程内核、Agent Worker 或正式 CLI 门禁的责任边界。

## 空间星仪

- Narrative Projection v3 提供稳定 revision、节点详情、语义边、LOD 摘要、决策、任务观察和项目进度的真实投影；
- PixiJS 场域支持全向平移、滚轮缩放、惯性、聚焦、六种可复现构型和 WebGL context-lost 的列表回退；
- 节点、关联线、DOM 标签和环境层使用远中近 LOD。大型作品先看主结构，再在聚焦时打开细节；
- 多个空间仪表可同时打开、拖动、调节尺寸、折叠、复位和按项目/构型持久化；展开窗口超过 12 个时，较早窗口会进入可恢复的最小化轨道；
- 顾问、节点详情、决策和阅读相关窗口统一为紧凑、可滚动的深矿物玻璃，不再因拖动失去既定比例。

## 创作与观测

- 推进、决策、规则、健康、正文、档案和交付均从正式 read model 与现有 API 读取，不伪造完成状态；空间与普通交付窗会启动受控的正式交付 Worker、刷新证据并下载已经生成的文件；
- Agent 任务观察通过实时投影显示当前角色、阶段、近期事件与正式进度，但不泄露内部推理；
- 节奏、详略、桥接和字数目标继续进入 compose、generate、review 与 longform audit 的正式链路；
- 设置新增场域与动效：完整、克制或静止动效；深度、平衡或平面阅读；自动、高质量或性能优先渲染。

## 兼容与升级

- v0.8 的经典脉络视图仍可在星仪控制中切回；切换不更改项目事实、当前任务或正式状态；
- 原有项目无需迁移。空间布局、窗口位置与视觉偏好只保存在 Studio 本地数据中；
- 原作品库、项目目录、模型凭证和桌面应用 identifier 保持兼容。

## 已验证与待验收

- 已完成：Python 回归、Vue 类型检查、Rust cargo check 与格式差异检查；
- 发布前仍应在干净 Windows 桌面环境使用 `packaging/build_desktop.ps1` 重建冻结 sidecar，并执行 Playwright/Canvas 像素截图、长篇性能采样、Tauri 冷启动、NSIS 全新安装与覆盖升级 smoke test。逐项操作见 [desktop acceptance checklist](v0.9.0-desktop-acceptance.md)，本次构建与签名状态见 [verification record](v0.9.0-verification.md)。
